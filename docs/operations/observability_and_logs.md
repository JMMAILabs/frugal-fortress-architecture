# Observability, Tracing, and FinOps Logging

The Frugal Fortress uses a modern, OpenTelemetry-native observability stack to ensure 100% visibility into both synchronous API requests and asynchronous background jobs.

## 1. The Observability Stack (Docker Compose)
*   **Logging:** `structlog` (JSON structured logging) -> Promtail -> **Loki**.
*   **Tracing:** OpenTelemetry (OTEL) SDK -> **Tempo**.
*   **Metrics:** Prometheus Client -> **Prometheus**.
*   **Visualization:** **Grafana**.

## 2. Distributed Trace Context Propagation
To debug distributed systems, a single request must be traceable across boundaries (API -> Database -> Redis -> Worker -> LLM).

1. **API Gateway:** `RequestIDMiddleware` generates a unique `X-Request-ID`.
2. **Context Binding:** The ID, along with the `tenant_id`, is bound to `structlog.contextvars` and the OTEL current span.
3. **The Asynchronous Boundary (Arq):** When the API enqueues a background job (e.g., `process_pdf_task`), the `trace_context` is explicitly serialized into the payload. The worker extracts this context, ensuring that logs generated by the background worker share the exact same `trace_id` as the original HTTP request.

## 3. FinOps Logging (The `FinOpsCallback`)
Every LLM interaction is intercepted by our custom `FinOpsCallback` (integrated with LiteLLM). Regardless of whether the call succeeds or fails, a structured log event (`finops_transaction`) is emitted containing:
*   `model_used`
*   `prompt_tokens` & `completion_tokens`
*   `cost_usd` (Calculated via `PricingRegistry`)
*   `latency_ms`
*   `tenant_id` & `trace_id`

This allows us to build real-time Grafana dashboards tracking our exact AWS/GCP/Groq spend per user, per module, enabling strict capacity planning.

## 4. Querying Logs in Grafana (Loki)
To investigate a failure (e.g., a failed PDF parse):
1. Open Grafana -> Explore -> Loki.
2. **Query by Trace ID:** `{job="app"} |= "your-trace-id-here"`
3. **Query by Tenant:** `{job="app"} | json | tenant_id="user-123"`
4. **Query by Error Level:** `{job="app"} | json | level="error" or level="critical"`
